# PostgreSQL & pgvector Best Practices

## Dependency Management
- We use `sqlx` as our database driver. It provides compile-time checked async SQL queries.
- Do NOT use heavy Object-Relational Mappers (ORMs) like Diesel. Write your SQL queries explicitly inside `sqlx::query!`.

## Managing Vectors
- The vector chunks generated by `fastembed-rs` should map to the `vector` type in PostgreSQL via the `pgvector` extension.
- Always use high-performance index types (like HNSW or IVFFlat) when establishing vector columns to ensure low-latency semantic search queries across memory nodes.
- Map the resulting structures in Rust precisely to arrays like `Vec<f32>` and properly bind them to `sqlx`.
